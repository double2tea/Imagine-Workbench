# 安全策略

[English](SECURITY.md) | [简体中文](SECURITY.zh-CN.md)

## 支持版本

`main` 分支是当前支持的开发线。

## 报告漏洞

请通过邮件私下报告安全问题：

```text
double_tea@foxmail.com
```

请包含：

- 受影响的路由、组件或工作流。
- 复现步骤。
- 影响评估。
- 相关日志或截图，并抹除其中的密钥。

问题完成初步分诊前，请不要公开利用细节。

## 安全说明

- 不要在公开部署中暴露供应商 API key。
- 对暴露 `/v1/*` 路由的公共或团队部署，请设置 `OPENAI_COMPAT_API_KEY`。
- 确保 `.env*`、本地构建输出和私有工作流目录不进入 Git。
- 生成素材和 board 存储在浏览器 IndexedDB；导出工作区 ZIP 时，如果包含凭证，用户应将其视为敏感文件。
